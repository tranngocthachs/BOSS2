Accessibility

BOSS2 attempts to be as accessible as possible to all users.

Web technologies

In order to be most accessible for users, BOSS2 followed the following guidelines during development:

  • BOSS2 tries to adhere to XHTML strict with standard CSS2.
  • BOSS2 tries to adhere to WAI to at least the first level.

If any page fails to validate against these standards, please contact the maintainers.

Javascript

BOSS2 uses Javascript for the following features:

  • Displaying a calendar picker for date/time input in the staff section.
  • Pop-up windows and forms. This is achieved by rewriting anchor and form tags to use the non-standard 'target' tag if possible. If not possible, links will display in situ.
  • Providing a 'Close Window' link in pages that were designed to be popped up.

User agent compatibility

BOSS2 has been tested with the following user agents:

  • Recommended browsers:
    • Firefox 1.5 onwards (or any post-2005 Gecko-based browser such as Epiphany or Camino)
    • Safari 1.0 onwards (or any post-2005 Webkit-based browser such as Konqueror 4.x or Google Chrome)
    • Internet Explorer 7 (note: cannot display calendar in staff frontend)
  • Other tested user agents:
    • Internet Explorer 6
    • Konqueror 3.5
    • Symbian Series 60 3rd Edition Web Browser
    • Opera 8 and Mini
    • Links 2
    • Lynx 2 (not recommended)
    • Netcat 1.10 (not recommended)
